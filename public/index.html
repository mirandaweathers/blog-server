  • GET /Users

GET /Users

input: none

output: JSON array of users currently in memory, hiding passwords + 200 OK status

error handling: if no users have been added, returns blank array

POST /Users

input: JSON object for user with all the following attributes:

  • {
    • "userId": "string",
    • "firstName": "string",
    • "lastName": "string",
    • "emailAddress": "string",
    • "password": "string"
  • }

output: JSON object with new user's information, except for password + 201 Created status

  • {
    • "userId": "string",
    • "firstName": "string",
    • "lastName": "string",
    • "emailAddress": "string"
  • }

error handling:

  • if not all user fields present, 406 Unacceptable status + message
  • if userId already exists, 409 Conflict status + message

GET /Users/{userId}

input: userId, given in URL

output: JSON object with user's information, except for password + 200 OK status

  • {
    • "userId": "string",
    • "firstName": "string",
    • "lastName": "string",
    • "emailAddress": "string"
  • }

error handling: if user not found, 404 Not Found status + error message

PATCH /Users/{userId}

input: userId, given in URL, plus JSON object for user with any or all of the following attributes:

  • {
    • "userId": "string",
    • "firstName": "string",
    • "lastName": "string",
    • "emailAddress": "string",
    • "password": "string"
  • }
*note: even if userId is changed within the request body, the new value will not be stored

output: if user found, 200 OK status + update and display updated user's information (password hidden) as JSON object:

  • {
    • "userId": "string",
    • "firstName": "string",
    • "lastName": "string",
    • "emailAddress": "string"
  • }

error handling: if user not found, 404 Not Found status + error message

DELETE /Users/{userId}

input: userId, given in URL

output: user is removed from array in memory and 204 No Content status

error handling: if user not found, 404 Not Found status + error message
